## üìÅ Demo Structure

Every demo in this repository follows a consistent four-folder structure:

### 1. üìù Prompt Templates
Contains the prompts used to interact with IBM Bob, including:
- Initial prompts that started the demo
- Follow-up prompts for iterative refinement
- Custom instructions or mode-specific settings

**Purpose:** Learn how to effectively communicate with Bob and reproduce similar results.

### 2. üó∫Ô∏è Implementation Journey Overview
Documents the step-by-step workflow, including:
- A markdown file chronicling the entire process
- Screenshots or recordings of key moments
- Challenges encountered and how they were solved
- Lessons learned and best practices

**Purpose:** Understand the decision-making process and learn from both successes and challenges.

**Note:** No PowerPoint files - we use markdown and visual media for better accessibility and version control.

### 3. üìÑ Input Documents
Example input files that Bob processes, such as:
- Source code files
- Configuration files
- Documentation
- Any other relevant materials

**Purpose:** Provide the starting point and context for the demo.

### 4. ‚ú® Bob Generated Output
The final artifacts produced by Bob, including:
- Created or modified code files
- Generated documentation
- Test files and validation results
- Any other output from the demo
- This folder is optional if the demo is not producing output files, or you think they are not worth to show

**Purpose:** Show the end result and demonstrate Bob's capabilities.

## üöÄ Getting Started

### Using the Template

1. Copy the [`Template Demo`](Template%20Demo/) folder
2. Rename it to describe your specific demo (e.g., "React Component Migration Demo")
3. Populate each folder with your demo-specific content
4. Update the demo's README.md with your specific details
5. Add your demo to the list below

### Creating Your Own Demo

Each demo should be:
- **Focused:** Address a specific use case or problem
- **Bite-sized:** Completable in a reasonable timeframe
- **Well-documented:** Include clear explanations and context
- **Reproducible:** Others should be able to follow along

## üí° Tips for Working with Bob

- **Choose the right mode:** Code, Plan, Ask, Advanced, or Orchestrator - each has its strengths
- **Be specific:** Clear, detailed prompts yield better results
- **Iterate:** Don't expect perfection on the first try - refine as needed
- **Document everything:** Capture your process for others to learn from
- **Share challenges:** Failed attempts are valuable learning opportunities
